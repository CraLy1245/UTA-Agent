# 第 1 阶段设计系统

视觉基准：`phase-1-chat-concept.png`。

## 方向

- 产品形态：完整桌面工具界面，不使用营销页或 CLI 作为入口。
- 主色：深海军蓝导航 `#071b31`、生存绿 `#198754`、真实白主画布。
- 信息密度：桌面三栏紧凑布局，状态信息可扫读，管理页面使用列表和表格。
- 图标：统一使用 Lucide 线性图标；交互必须同时具备文本或可访问名称。

## 布局

- 桌面：344px 会话栏、弹性对话区、326px 状态栏。
- 中等宽度：两侧栏缩窄；900px 以下自动进入图标导航模式。
- 620px 以下：隐藏右侧状态栏，保留 54px 图标导航和单列内容。
- 会话栏与状态栏支持用户手动折叠，并为后续状态持久化保留独立 Store。

## 组件边界

- `ConversationSidebar`：品牌、主导航、会话检索与会话列表。
- `ChatPage`：消息、工具状态、反馈与输入区。
- `AgentSidebar`：Token、记忆、整理、Skill 与后台任务状态。
- `PageScaffold`：记忆、Skill、活动和设置页面的共同标题与内容框架。

第 1 阶段的数据与交互均为 Mock；第 2 阶段接入 API 时保持这些展示组件与数据访问层解耦。
