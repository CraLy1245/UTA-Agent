# 安全策略

## 支持范围

Survival Agent 当前是实验性 `0.1.0-alpha` 项目，没有生产级安全支持承诺。仅最新 `main` 分支接受安全修复；旧提交、个人 Fork 和第三方安装包不在支持范围内。

## 私下报告漏洞

请使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口。不要为未修复漏洞创建公开 Issue。

报告中请包含受影响版本、复现步骤、影响、最小证明和建议修复。请先移除 API Key、代理凭据、个人数据、数据库、日志中的敏感内容以及本机绝对路径。

维护者会尽力确认报告，但目前不承诺固定响应时间、漏洞赏金或发布时限。

## 用户安全边界

- 只从可信来源构建或下载应用；当前 Windows 安装包未签名。
- 使用你信任的 OpenAI 兼容 Provider，并独立核对其隐私、计费和数据保留政策。
- 开发密钥仅放在进程环境变量；桌面密钥仅放在 Windows 凭据管理器。
- Workspace 应使用专门目录，不要指向用户主目录、磁盘根目录或重要项目目录。
- 公开错误报告前先检查日志、导出和截图，避免泄露对话及文件内容。

如果密钥曾出现在聊天、终端历史、日志或提交中，应立即在服务商处撤销并重新生成；仅从文件中删除并不能使旧密钥重新安全。
