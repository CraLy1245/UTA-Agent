# 路线图

本路线图描述公开实验版本之后的方向，不构成发布时间承诺。完成状态以 [`docs/development-status.md`](docs/development-status.md) 和实际测试为准。

## 发布前阻塞项

- [ ] 在桌面设置页使用用户自行保存的凭据完成最终真实模型流式复验。
- [ ] 重新打包并验证包含最新 Cognitive Worker 输出契约修复的安装包。
- [ ] 为 Windows 可执行文件和 NSIS 安装包配置可信代码签名。
- [ ] 建立可重复的 GitHub Actions 检查，覆盖前端、后端、迁移与 Rust 测试。
- [ ] 建立正式 Release 校验和、构建来源和升级/回滚说明。

## 稳定化方向

- [ ] 将 Provider 差异适配与模型能力探测从业务逻辑中进一步隔离。
- [ ] 增加认知整理 JSON 失败的可观察纠错流程和费用上限。
- [ ] 增加数据库备份恢复向导、迁移失败回滚和损坏诊断界面。
- [ ] 完成工具权限审计、恶意路径测试和第三方安全审查。
- [ ] 明确长期版本兼容、数据库 Schema 支持周期和弃用政策。

## 暂不承诺

远程同步、多用户、自动更新、插件市场、macOS/Linux 桌面端和生产级 SLA 尚未进入承诺范围。实现这些能力前应先稳定本地数据模型和安全边界，避免过早锁定架构。
